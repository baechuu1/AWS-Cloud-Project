# 🔍 Cloud Reconnaissance

## 📌 Overview
This phase simulates an attacker using valid IAM access keys from an external system (Kali Linux) to enumerate AWS resources via API calls.

The objective is to understand how attackers perform reconnaissance in cloud environments and how this activity is captured in AWS CloudTrail.

---

## ⚔️ Attack Simulation

The attacker performs API-based enumeration using AWS CLI:

```bash
aws s3 ls
aws iam list-users
aws iam list-roles
aws sts get-caller-identity
```
📥 Log Collection

All API activity is recorded in AWS CloudTrail.

CloudTrail logs include:

Event name (API call)
User identity
Source IP address
Timestamp
Error codes (if applicable)
🔍 Detection & Analysis
Key Observations
Successful enumeration of S3 buckets
Attempts to enumerate IAM resources
Multiple AccessDenied responses
API activity originating from external system
🚨 Detection Opportunities
API activity from unusual IP addresses
Enumeration behavior (ListUsers, ListRoles, ListBuckets)
Repeated AccessDenied events
Use of access keys instead of console login

🧬 MITRE ATT&CK Mapping
Technique	Description
Valid Accounts	Use of legitimate IAM credentials
Account Discovery	Enumerating IAM users
Cloud Service Discovery	Listing AWS resources
🛡️ Mitigation & Defense
Enforce least privilege IAM policies
Monitor API activity using CloudTrail
Alert on repeated AccessDenied events
Restrict and rotate access keys
Implement centralized logging and SIEM monitoring
📌 Summary

This phase demonstrates how attackers use valid credentials to perform reconnaissance and how this behavior can be detected through API activity monitoring.
